# Trading UI Designer Skill

이 스킬은 고성능 모듈형 자동매매 시스템을 위한 웹 기반 사용자 인터페이스(UI)를 설계하고, 백엔드와의 실시간 데이터 연동 방안을 수립하는 데 필요한 지침과 자원을 제공합니다.

## 1. 주요 기능 및 활용 시점

*   **UI/UX 설계**: 실제 거래소 수준의 전문적인 웹 트레이딩 대시보드 디자인을 구상할 때.
*   **기술 스택 선정**: 웹 프론트엔드(React, Vue), 차트 라이브러리(TradingView), 백엔드(FastAPI) 등 최적의 기술 스택을 결정할 때.
*   **실시간 데이터 흐름 분석**: 웹소켓을 통한 차트, 호가, 잔고 데이터의 수집, 분배, 프론트엔드 동기화 과정을 이해하고 설계할 때.
*   **시퀀스 다이어그램 생성**: 복잡한 데이터 흐름을 시각적으로 명확하게 표현해야 할 때.

## 2. 스킬 활용 가이드

### 2.1. UI 구현 방식 비교

PyQt와 같은 데스크톱 앱과 웹 기반 UI의 장단점을 비교하여 프로젝트에 가장 적합한 방식을 선택합니다. 일반적으로 **화려하고 동적인 실시간 차트와 높은 접근성을 원한다면 웹 기반을 추천**합니다.

### 2.2. 웹 기반 UI 기술 스택 참조

[tech_stack.md](references/tech_stack.md) 파일을 참조하여 웹 기반 트레이딩 UI 구현에 필요한 핵심 기술 스택(프레임워크, 차트 라이브러리, 통신 프로토콜 등)에 대한 정보를 얻을 수 있습니다.

### 2.3. 실시간 데이터 흐름 패턴 이해

[data_flow_patterns.md](references/data_flow_patterns.md) 파일을 참조하여 웹소켓을 통한 실시간 데이터(차트, 호가, 잔고)의 동적 관리 및 효율적인 흐름 패턴을 이해하고 설계에 적용합니다.

### 2.4. 시퀀스 다이어그램 생성

데이터 흐름을 명확히 시각화하기 위해 Mermaid 문법을 사용하여 시퀀스 다이어그램을 생성할 수 있습니다. `manus-render-diagram` 유틸리티를 사용하여 `.mmd` 파일을 `.png` 이미지로 변환할 수 있습니다.

```bash
manus-render-diagram path/to/your_diagram.mmd path/to/output.png
```

## 3. 스킬 사용 예시

사용자가 '자동매매 시스템의 웹 UI 디자인을 보여줘' 또는 '실시간 차트 데이터가 어떻게 흘러가는지 설명해줘'와 같은 요청을 할 때 이 스킬을 활성화하여 관련 정보를 제공하고 디자인 이미지, 다이어그램 등을 생성할 수 있습니다.
