# 자동매매 시스템 웹 UI 디자인 상세 설명

생성된 웹 대시보드 UI 디자인은 **[고성능 모듈형 자동매매 시스템 아키텍처]**의 핵심 기능을 사용자가 직관적으로 제어하고 모니터링할 수 있도록 설계되었습니다.

## 1. 디자인 컨셉
*   **테마**: 전문 트레이딩 환경에 최적화된 **다크 모드(Dark Mode)**를 적용하여 눈의 피로도를 낮추고 데이터 가독성을 극대화했습니다.
*   **컬러 스킴**: 상승(Green), 하락(Red), 주요 지표(Blue/Purple)를 명확히 구분하여 실시간 변화를 즉각적으로 인지할 수 있게 했습니다.

## 2. 주요 UI 영역 및 기능

### 2.1. 실시간 멀티 차트 (중앙)
*   **TradingView 스타일**: 캔들스틱 차트와 함께 거래량, RSI, MACD 등 기술적 지표가 실시간으로 렌더링됩니다.
*   **인터랙티브 기능**: 타임프레임 변경, 지표 추가/제거, 차트 드로잉 도구 등을 포함하여 전문적인 분석이 가능합니다.

### 2.2. 종목별 독립 스레드 모니터 (왼쪽)
*   **Active Threads**: 현재 실행 중인 각 종목별 스레드(BTC, ETH, 삼성전자 등)의 상태를 실시간으로 보여줍니다.
*   **상태 표시**: 'LIVE' 표시등을 통해 스레드의 정상 작동 여부를 확인하고, 실시간 수익률(P/L)을 즉시 파악할 수 있습니다.

### 2.3. 동적 전략 설정 패널 (오른쪽)
*   **전략 다중 선택**: RSI, Moving Average, Bollinger Bands 등 다양한 전략을 스위치로 켜고 끌 수 있습니다.
*   **파라미터 조정**: 각 전략의 기간(Period), 기준값 등을 슬라이더와 입력창을 통해 실시간으로 조정하여 즉시 적용할 수 있습니다. 이는 아키텍처의 **'Composite Strategy'** 기능을 시각화한 것입니다.

### 2.4. 실시간 데이터 및 주문 내역 (하단)
*   **Order Book (호가창)**: 매수/매도 잔량을 시각화하여 시장의 수급 상황을 실시간으로 파악합니다.
*   **Real-time Logs**: 시스템 내부에서 발생하는 이벤트(신호 발생, 리스크 체크 통과 등)를 실시간으로 출력합니다.
*   **Execution History**: 실제 체결된 주문 내역을 테이블 형태로 보여줍니다.

### 2.5. 시스템 헬스체크 (좌측 하단)
*   API 연결 상태, 데이터 수신 속도(Latency), 가동 시간(Uptime) 등을 모니터링하여 시스템의 안정성을 실시간으로 확인합니다.

## 3. 기술적 구현 제언
*   **Frontend**: React.js와 Tailwind CSS를 사용하여 컴포넌트 기반으로 구축.
*   **Chart**: TradingView Lightweight Charts 라이브러리 활용.
*   **Real-time**: WebSocket을 통해 백엔드 파이썬 엔진과 통신하여 0.1초 단위의 데이터 업데이트 구현.
