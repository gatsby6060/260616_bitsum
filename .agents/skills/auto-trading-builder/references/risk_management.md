# 리스크 관리 기법 레퍼런스

자동매매 시스템에서 사용하는 리스크 관리 기법 전체 목록과 설계 원칙을 정의한다.
모든 리스크 관리자는 `BaseRiskManager`를 상속받고, `templates/risk_managers.py`에 구현체가 있다.

---

## 리스크 관리 4계층

```
① 포지션 레벨   — 개별 종목의 손익 제어
② 포트폴리오 레벨 — 전체 자산의 드로우다운 제어
③ 주문 실행 레벨 — 단일 주문 금액·수량 제한
④ 운영 레벨     — 일일 손실 한도, API 오류 처리
```

---

## 1. 포지션 레벨 리스크

### 1-1. 고정 손절/익절 (StopLossRiskManager)

진입가 대비 고정 비율에서 강제 청산한다. 가장 기본적인 기법.

```
매수가: 10,000원
손절선: 9,700원 (−3%)  → 이탈 시 강제 SELL
익절선: 10,600원 (+6%) → 도달 시 SELL
```

설정 키: `stop_loss_pct`, `take_profit_pct`

---

### 1-2. 트레일링 스탑 (TrailingStopRiskManager)

최고가를 추적하며 손절선을 동적으로 상향 조정한다.
추세 추종 전략과 조합할 때 효과적이다.

```
최고가 11,500원 → 손절선: 11,155원 (−3%)
최고가 12,000원 → 손절선: 11,640원 (−3%)  ← 자동 상향
가격 11,500원 하락 → 손절선(11,640) 이탈 → SELL
```

설정 키: `trail_pct`

---

### 1-3. 트레일링 익절 (TrailingTakeProfitRiskManager) ★

익절 목표 도달 후 즉시 청산하지 않고 트레일링 모드로 전환한다.
추세가 지속되면 이익을 극대화하고, 추세가 꺾이면 청산한다.

```
상태 전환:
  [WATCHING] → 익절 목표(+6%) 도달 → [TRAILING]
  [TRAILING] → 최고가 대비 −N% 하락 → SELL

예시:
  진입  10,000원
  목표  10,600원 도달 → 트레일링 모드 전환
  12,000원 → 추적 중 (최고가 갱신)
  13,500원 → 추적 중 (최고가 갱신)
  12,800원 (−5%) → SELL → 수익 +28%
```

설정 키: `stop_loss_pct`, `take_profit_pct`(트리거), `trail_pct`

> **고급 변형 — 단계별 동적 트레일**
> 수익률이 높아질수록 트레일 간격을 좁혀 이익을 강하게 보호한다.
> `profit_pct < 0.15` → trail_pct=0.05 / `profit_pct >= 0.30` → trail_pct=0.02

---

### 1-4. 시간 기반 손절 (TimedExitRiskManager)

일정 시간 이상 보유 시 수익 무관하게 청산한다. 기회비용을 방지한다.

```
24시간 보유 초과 → SELL (횡보장 대응)
```

설정 키: `max_hold_hours`

---

## 2. 분할 매수 전략 (물타기 / 불타기)

### 2-1. 물타기 — 평균단가 낮추기 (AveragingDownRiskManager)

손실 중인 포지션에 추가 매수하여 평균 진입가를 낮춘다.
횡보장·반등 기대 시 사용한다.

```
진입:      10,000원 × 100만원  → 평균단가 10,000원
−5% 하락:   9,500원 ×  50만원  → 평균단가  9,770원
−10% 하락:  9,000원 ×  25만원  → 평균단가  9,634원

최종 손절선(hard_stop): 진입가 × (1 − 0.20) = 8,000원 → 전량 청산
```

반환 신호: `ADD_BUY` (추가 매수) / `HARD_STOP` (전량 청산)

설정 키:
- `drop_trigger_pct`: 몇 % 하락마다 추가 매수 (예: 0.05)
- `max_add_count`: 최대 추가 매수 횟수 (예: 3)
- `add_ratio`: 직전 투자금 대비 추가 비율 (예: 0.5 → 절반씩)
- `hard_stop_pct`: 최종 손절선 (예: 0.20 → −20%)

> ⚠️ **필수 규칙**: `hard_stop_pct` 없는 물타기는 계좌 폭발 위험.
> 예비 자금을 반드시 확보해두고 사용할 것.

---

### 2-2. 불타기(피라미딩) — 추세 이익 극대화 (PyramidingRiskManager)

수익 중인 포지션에 추가 매수하여 추세 이익을 극대화한다.
강한 상승 추세 시 사용한다. 트레일링 스탑과 반드시 세트로 사용한다.

```
진입:     10,000원 × 100만원  비중 ████████░░
+5% 상승: 10,500원 ×  50만원  비중 ████░░░░░░  (역피라미드: 절반씩 줄임)
+10% 상승:11,025원 ×  25만원  비중 ██░░░░░░░░

최고가 11,500원 → 트레일링 스탑(−3%) = 11,155원
가격 11,155원 이탈 → SELL (전량 청산)
```

반환 신호: `ADD_BUY` (추가 매수) / `SELL` (트레일링 스탑 청산)

설정 키:
- `rise_trigger_pct`: 몇 % 상승마다 추가 매수 (예: 0.05)
- `max_add_count`: 최대 추가 매수 횟수 (예: 3)
- `add_ratio`: 직전 투자금 대비 추가 비율 (예: 0.5)
- `trail_stop_pct`: 트레일링 스탑 간격 (예: 0.03 → −3%)

> **역피라미드 원칙**: 추가 매수 금액을 점차 줄여야(절반씩) 평균 진입가 상승을 억제한다.

## 3. 포트폴리오 레벨 리스크

### 3-1. 최대 드로우다운 제한 (DrawdownGuard)

전고점 대비 손실이 한도를 초과하면 전체 거래를 중단한다.

```
설정: max_drawdown = 20%
전고점: 1,200만원
현재:     900만원 → 드로우다운 −25% → 거래 중단
```

설정 키: `max_drawdown_pct`

---

### 3-2. 포지션 사이징 (PositionSizer)

한 종목에 투자할 금액을 전체 자산 대비 비율로 결정한다.

| 기법 | 공식 | 특징 |
|------|------|------|
| 고정 비율 | `투자금 = 자산 × ratio` | 단순, 예측 가능 |
| 켈리 공식 | `f = (b×p − q) / b` | 승률 기반 최적 비율 |
| 변동성 기반 | `투자금 = 목표위험 / ATR` | 변동성 큰 종목 자동 조절 |

> **Half Kelly**: 켈리 공식 결과의 50%만 사용하여 과레버리지 방지 권장.

---

## 4. 운영 레벨 리스크

### 4-1. 일일 손실 한도 (DailyLossGuard)

하루 시작 잔고 대비 손실이 한도를 넘으면 당일 거래 중단.

설정 키: `daily_loss_limit_pct` (예: 0.05 → −5%)

---

### 4-2. API 재시도 (지수 백오프)

주문 전송 실패 시 최대 3회 재시도, 간격은 2의 지수 배로 증가.

```python
for attempt in range(3):
    try:
        result = api.place_order(...)
        break
    except Exception:
        time.sleep(2 ** attempt)  # 1s → 2s → 4s
```

## ADD_BUY 신호 처리 규칙

물타기·불타기는 기존 `BUY/SELL/HOLD` 외에 `ADD_BUY` 신호를 반환한다.
`OrderExecutor`는 이 신호를 받아 아래 로직으로 처리한다.

```python
if signal == "ADD_BUY":
    add_amount  = position["invested_amount"] * add_ratio
    new_qty     = add_amount / current_price
    total_qty   = position["quantity"] + new_qty
    total_cost  = position["total_cost"] + add_amount
    position["avg_price"] = total_cost / total_qty  # 평균단가 재계산 필수
    position["add_count"] += 1
```

## 전략별 리스크 관리자 조합 추천

| 시장 상황 | 전략 | 리스크 관리자 |
|-----------|------|--------------|
| 횡보장 | RSI, 이격도 | `AveragingDown` + 하드스탑 |
| 강한 상승 추세 | MACD | `Pyramiding` + 트레일링 |
| 추세 불명확 | RSI AND MACD | `TrailingTakeProfit` |
| 고변동성 장 | 볼린저밴드 | `StopLoss` (타이트) + `TimedExit` |

## 리스크 관리자 우선순위

`TickerWorker._process()` 내에서 아래 순서로 평가한다.

```
1순위: DailyLossGuard    — 일일 한도 초과 시 즉시 거래 중단
2순위: DrawdownGuard     — 드로우다운 한도 초과 시 거래 중단
3순위: RiskManager.check() — 포지션 레벨 손절/익절/추가매수 판단
4순위: 전략 신호         — BUY/SELL/HOLD
```
